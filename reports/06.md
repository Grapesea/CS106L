这里让Claude总结一下C++23出现的新特性：monadic function(单子操作)

C++23 里 `std::optional` 有3个monadic function：

| 方法 | 签名 | 作用 |
|---|---|---|
| [`transform`](https://en.cppreference.com/w/cpp/utility/optional/transform.html) | `optional<T>` → `optional<U>` | lambda 返回普通值 `U` |
| [`and_then`](https://en.cppreference.com/w/cpp/utility/optional/and_then.html) | `optional<T>` → `optional<U>` | lambda 返回 `optional<U>`，用于可能失败的链式操作 |
| [`or_else`](https://en.cppreference.com/w/cpp/utility/optional/or_else.html) | `optional<T>` → `optional<T>` | 无值时提供备选的 `optional<T>` |

这三个的共同特征是：**输入是 `optional`，输出还是 `optional`**，可以一直链式下去.

* `.and_then`: Calls a function to produce a new optional if there is a value; otherwise, returns nothing. The function passed to `and_then` takes a non-optional instance of type `T` and returns a `std::optional<U>`.
    
    If the optional has a value, `and_then` applies the function to its value and returns the result.
    
    If the optional doesn't have a value (i.e. it is `std::nullopt`), it returns `std::nullopt`.

    ```cpp
    template <typename U> std::optional<U> std::optional<T>::and_then(std::function<std::optional<U>(T)> func);
    ```

* `.tranform`: Applies a function to the stored value if present, wrapping the result in an optional, or returns nothing otherwise. The function passed to `transform` takes a non-optional instance of type `T` and returns a non-optional instance of type `U`.
    
    If the optional has a value, `transform` applies the function to its value and returns the result wrapped in an `std::optional<U>`.
    
    If the optional doesn't have a value (i.e. it is `std::nullopt`), it returns `std::nullopt`.

    ```cpp
    template <typename U> std::optional<U> std::optional<T>::transform(std::function<U(T)> func);
    ```

* `.or_else`: Returns the optional itself if it has a value; otherwise, it calls a function to produce a new optional. 是`and_then`相反的逻辑.

    The function passed to `or_else` takes in no arguments and returns a `std::optional<U>`.
    
    If the optional has a value, `or_else` returns it.
    
    If the optional doesn't have a value (i.e. it is `std::nullopt`), `or_else invokes the function and returns the result.
 
    ```cpp
    template <typename U> std::optional<U> std::optional<T>::or_else(std::function<std::optional<U>(T)> func);
    ```

而`value_or` 输出的是一个**裸的 `T`**，链就到此为止了，所以它是终止操作，不是 monadic operation.
