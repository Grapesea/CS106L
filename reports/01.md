# lab01

这份实验是相对简单的，但是坑却不少.

首先是`utils.cpp`文件中给出了`split`和删除的两个函数，这在之后会被调用；`split`的输出类型需要注意，一开始理解成`number_of_units`是int了.

其次是`getline`函数的调用（[std::getline - cppreference.com](https://en.cppreference.com/w/cpp/string/basic_string/getline)）忘记了，是`getline(csv_data, line);`本身已经有将`csv_data`中的一行传入`line`中的过程，不需要再用一个参数来承接.

然后就是重量级的两句格式问题了：

1. `if (p.quarter != "null\r")`这里不能写`null\n`，测试发现错误，原因：`std::getline` 以 `\n` 作为分隔符来切割行，读到 `\n` 就停止，并且**把 `\n` 丢弃**，而源csv文件是：

    ```text
    文件内容（十六进制）:
    ..., n, u, l, l, \r, \n, ...
                          ↑        ↑
                   残留在string   被getline丢弃
    ```

    因此写`null\n`是不可行的，因为根本不存在.

    据搜索，硬编码`null\r`其实是一个脆弱的方案，因为在Linux机器上`\r`是不存在的，所以还建议在这行之前再加上一句
    
    ```cpp
    if (!line.empty() && line.back() == '\r')
        line.pop_back(); // 统一去掉 \r，之后就能直接比较 "null"
    ```

2. `file << p.title << "," << p.number_of_units << "," << p.quarter;`如果直接将Course这个数组的元素传入file中，打印出来的格式类似：`Incentives in Computer Science, 3, 2023-2024 Spring`，但题目要求的是`Incentives in Computer Science,3,2023-2024 Spring`，中间少两个空格，所以需要每个参数拆开来传入.
