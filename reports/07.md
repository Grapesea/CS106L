[unique_ptr](https://en.cppreference.com/w/cpp/memory/unique_ptr.html)

这个部分的unique_ptr第一次接触感觉很迷，主要是内存释放折腾得一塌糊涂.

在AI的debug提示之下终于结束了：

```cpp
template <typename T> cs106l::unique_ptr<ListNode<T>> create_list(const std::vector<T>& values) {
  /* STUDENT TODO: Implement this method */
  cs106l::unique_ptr<ListNode<T>> head = nullptr;

  for (int i = values.size() - 1; i >= 0; i--){
    auto node = cs106l::make_unique<ListNode<T>>(values[i]);
    node->next = std::move(head);
    head = std::move(node);
  } 
  return head;
}
```

这里lab的提示中有一些注意点：

* `int i` 而不是 `size_t i`，因为如果小于0会出现未定义行为，直接报错.

* `make_unique`这个函数的功能是：把 `new` 和 `unique_ptr` 的构造合并成一步，确保裸指针从一开始就被 `unique_ptr` 管理，没有任何泄漏的机会

    ```cpp
    template <typename T, typename... Args> 
    unique_ptr<T> make_unique(Args&&... args) {
        return unique_ptr<T>(new T(std::forward<Args>(args)...));
    }
    ```

    在我们使用的时候，要做的声明是：

    ```cpp
    auto node = cs106l::make_unique<ListNode<T>>(values[i]);
    ```
