还是老生常谈的一个问题，member function需要在做之前读一遍看看有没有给出可以用的东西，比如此处的add_friend.

友元函数`friend std::ostream& operator<<(std::ostream& os, const User& user)`使用`std::ostream`而不是void直接输出，有缓冲区对整体书写逻辑也有一定的帮助.

还有一个要记忆下的是：

```cpp
User(User&& user) = delete;             // 禁止 move construction
User& operator=(User&& user) = delete;  // 禁止 move assignment
//设置函数为delete是为了防止右值操作.
```

禁止函数的操作是赋一个delete.
